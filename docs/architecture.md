# Архитектура системы

## Общая схема
Проект разделен на две независимые части:
1. **Backend (Python, FastAPI)**: Источник данных и логический центр.
2. **Frontend (TypeScript, Next.js)**: Визуализация и интерфейс пользователя.

## Стек технологий
- **Backend**: FastAPI, Pydantic, OpenAI API, python-dotenv.
- **Frontend**: Next.js 15 (App Router), React, Recharts (планируется), Lucide React (иконки), CSS Modules.

## Структура папок
### Backend
- `app/api/`: Обработка маршрутов (траспорт, экология, алерты и т.д.).
- `app/services/`: Бизнес-логика (генерация мок-данных, запросы к OpenAI).
- `app/schemas/`: Модели Pydantic для валидации данных.
- `app/data/`: Конфигурация сценариев (`scenarios.json`).

### Frontend
- `src/app/`: Основные страницы и маршруты.
- `src/components/`: Многоразовые UI-компоненты (карточки, панели).
- `src/lib/`: Утилиты и клиент для API бэкенда.
- `src/styles/`: Глобальные стили.

## Логика работы AI
При запросе `/api/ai-insights`, бэкенд собирает текущее состояние по активному сценарию и формирует промпт для OpenAI GPT-4o-mini. Ответ возвращается в структурированном JSON, содержащем сводку, уровень критичности и рекомендации. В случае отсутствия API-ключа предусмотрен механизм заглушек (fallback), имитирующий работу ИИ.
